Detailed Instructions for LF-mc

This page explains how to run the artifact for the QEST+FORMATS 2026 tool paper "LF-mc: An Efficient Verifier for Lingua Franca", how to reproduce the reported experiments, and how to build the tool components from source.

Archival artifact record: https://doi.org/10.5281/zenodo.19568831.

Runtime Docker image: ghcr.io/symbolicsafety/lf-mc:v1.2.

Required Software

Docker is required to run the artifact in the intended reproducible environment. Docker Desktop is recommended for macOS and Windows users. Linux users can install Docker Engine using their distribution's package manager or the official Docker instructions.

Windows users should run the provided shell scripts from WSL with a Linux distribution.

Docker Setup

Pull the artifact image from GitHub Container Registry:

user@host:~$ docker pull ghcr.io/symbolicsafety/lf-mc:v1.2

Start an interactive shell inside the container:

user@host:~$ docker run -it --rm ghcr.io/symbolicsafety/lf-mc:v1.2 bash -l

The image contains the LF compiler extension, the Maude runtime, the benchmark inputs, hyperfine, and helper tools used by the replication script.

Smoke Test

After entering the container, run LF-mc on the ADAS example from the paper:

docker:/# lfc /opt/lf-maude/examples/src/ADASModel.lf

This invokes LF-mc on an annotated Lingua Franca file, generates the corresponding Maude model and analysis commands, invokes Maude, and prints the simulation, reachability, and LTL results. The expected output includes successful analyses, one bounded LTL counterexample for the 54 ms bound, and a "No Solution" result for the bad-state reachability query.

docker:/# exit

Experiment Replication

Table 2 of the paper compares LF-mc with LF-Verifier and PolyVer on the 22 LF-Verifier benchmark programs. This artifact reruns the LF-mc column using the Docker image above. The LF-Verifier and PolyVer columns are comparison baselines from the published results of those tools.

The default run uses one hyperfine measurement and no warmup, which is intended as a quick artifact-evaluation run. A more stable timing run can use more repetitions.

Download and extract experiments.zip. It contains run-all-experiments.sh and tests.csv in the archive root. Run the commands below from the extracted directory.

user@host:~$ unzip experiments.zip

Default Run

user@host:experiments$ sh run-all-experiments.sh --overwrite

The script starts one Docker container, runs all benchmark programs listed in tests.csv, prints a timing table, and writes:

  • results.csv: CSV summary of generation, solving, and total time.
  • hyperfine-report.json: combined hyperfine data.
  • logs/: LF-mc output and per-benchmark hyperfine JSON files created during a run.

Stable Timing Run

user@host:experiments$ WARMUP=1 RUNS=20 sh run-all-experiments.sh --overwrite

Image Override

The script defaults to ghcr.io/symbolicsafety/lf-mc:v1.2. Override IMAGE if you are testing a locally built or retagged image:

user@host:experiments$ IMAGE=lf-mc-local:v1.2 sh run-all-experiments.sh --overwrite

Running Individual Benchmarks

The second column of tests.csv in experiments.zip gives the LF source path inside the Docker image. For example:

user@host:~$ docker run -it --rm ghcr.io/symbolicsafety/lf-mc:v1.2 bash -l
docker:/# lfc /opt/lf-maude/examples/src/TrainDoor.lf

The benchmark files in /opt/lf-maude/examples/src/ are useful templates for new inputs. In particular, ADASModel.lf shows simulation, reachability, LTL, and @maudePhysAct annotations in one file.

Using LF-mc on New Inputs

LF-mc reads @maude and @maudePhysAct annotations from an LF source file, generates the Maude model and analysis commands, invokes Maude, and prints the results.

You can edit files inside the container for quick experiments, or mount a host directory containing LF files:

user@host:inputs$ docker run -it --rm -v "$PWD:/work" ghcr.io/symbolicsafety/lf-mc:v1.2 bash -l
docker:/# lfc /work/MyModel.lf

For mounted host inputs, generated files are written relative to the mounted input directory when permitted by the LF compiler.

Building From Source

The source code is available in the local archive code.zip, through the archived artifact record, and from the corresponding GitHub releases: lf-maude v1.2.2 and lf-mc v1.2. The archived artifact source tree has this structure:

artifact root/ ├── experiments.zip ├── code/ │ ├── lf-maude-v1.2.2/ │ └── lf-mc-v1.2/ └── supporting archives and results

Maude Backend

code/lf-maude-v1.2.2/ contains the Maude backend/runtime used by LF-mc. The root contains the Maude files implementing the LF interpreter. Its examples/src/ directory contains annotated LF programs, and examples/mod-gen/ contains generated Maude models for bundled examples.

LF Compiler Extensions

code/lf-mc-v1.2/ contains the Lingua Franca compiler extension that automatically translates .lf files with analysis annotations into Maude files that can be run by lf-maude.

code/lf-mc-v1.2/ ├── ... ├── core │ └── src │ └── main │ ├── antlr │ │ ├── LTLLexer.g4 │ │ ├── LTLParser.g4 │ │ └── ... │ └── java │ └── org │ └── lflang │ ├── analyses │ │ ├── c │ │ │ ├── CToMaudeVisitor.java │ │ │ └── ... │ │ └── maude │ │ ├── LTLVisitor.java │ │ ├── MaudeGenerator.java │ │ └── ... │ └── ... └── ...

This is a Java project built using Gradle. The build used for this submission is included in the Docker image referenced above. A standalone LF-mc package is also available here for users who want to run LF-mc without building the compiler extension from source: lf-mc-v1.2.tar.gz.

user@host:artifact$ tar -xzf lf-mc-v1.2.tar.gz

The archive extracts to lf-cli-lf-mc-v1.2-lf-base-v0.9.1/ and includes command-line binaries such as bin/lfc, bin/lff, and bin/lfd. Its SHA-256 checksum is:

232c719cbbb3a2e6a5ed8d4024b31939f77122fbaa527ecb39c4f289f2819b72 lf-mc-v1.2.tar.gz

To build the project from source instead, run ./gradlew assemble at the root of the code/lf-mc-v1.2 folder:

user@host:code/lf-mc-v1.2$ ./gradlew assemble

Building from source requires a Java/Gradle environment and a machine with substantial memory; the artifact notes recommend more than 16 GB of RAM.
